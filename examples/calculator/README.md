# Calculator

This example demonstrates the use of GoLR for building a simple integer-based calculator.

The calculator accepts a single arithmetic expression as a command-line argument and prints the result:

```sh
calculator "4 + 5 * 3"
# 19
```

It supports:

- The four basic operators `+`, `-`, `*`, `/` with standard mathematical precedence (`*` and `/` bind tighter than `+` 
  and `-`)
- Left-to-right associativity for all operators (10 - 3 - 2 evaluates to 5, not 9)
- Unary minus (-5, -2 * 3)
- Grouping with parentheses ((2 + 3) * 4)
- Integer arithmetic (negative results are allowed; division truncates toward zero)
- Whitespace between tokens is ignored

## Structure

```text
spec/calculator.golr   Grammar file describing the language
parser/                Scanner and parser generated from the grammar (do not edit)
main.go                CLI entry point and tree-walking evaluator
main_test.go           Tests for the evaluator
```

### Grammar (spec/calculator.golr)

The grammar is the starting point. The @scanner section defines the tokens (integers, operators, parentheses), and the
@parser section defines the grammar productions. A virtual token UMINUS is declared with @empty — it is never matched
by the scanner but is used purely as a precedence marker to make unary minus bind tighter than * and /.

### Generated parser (parser/)

The files in `parser/` are generated by running GoLR against the grammar file. They should not be edited by hand. The
generated code provides a Scanner that tokenizes input, a TokenSkipper that discards whitespace tokens, and a Parser
that builds a parse tree.

### Evaluator (main.go)

`evaluateNode` walks the parse tree recursively. Because each grammar production has a distinct number of children, the
shape of the node tells the evaluator which production was matched:

| Children | Production                                                                  |
|----------|-----------------------------------------------------------------------------|
| 1        | INTEGER literal                                                             |
| 2        | Unary minus: "-" expression                                                 |
| 3        | Binary operation: expression OP expression, or grouping: "(" expression ")" |

For the 3-child case, checking whether the middle child is a terminal distinguishes a binary operation from a grouped
sub-expression.

## Running the Example

```shell
# Generate the scanner and parser from the grammar (re-run whenever the grammar changes)
golr scanner --frontend golr --frontend-file-path examples/calculator/spec/calculator.golr --backend go --backend-file-path examples/calculator/parser/scanner.go
golr parser --frontend golr --frontend-file-path examples/calculator/spec/calculator.golr --backend go --backend-file-path examples/calculator/parser/parser.go

# Build and run
go run ./examples/calculator "2 + 3 * 4"
# 14

# Run the tests
go test ./examples/calculator
```
